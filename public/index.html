Overview

JavaScript Development (JSD) is a ten-week course covering the programming language JavaScript: the native language of the web used by developers the world over. JSD is comprised of a total of 60 hours of in-class instruction spread evenly over 20 classes; it breaks down into four units:


1.

Fundamentals of JavaScript

2.

The Browser and Application Programming Interfaces (API's)

3.

Persisting Data and Advanced Topics

4.

Building and Deploying Your App

Pre-Course Checklist

JSD is challenging! To ensure you are ready for what lies ahead, it is HIGHLY RECOMMENDED that you spend, at bare minimum, six-to-ten hours acquainting yourself with some of the core technologies we will be using throughout the course.

As JSD assumes students possess a working knowledge of HTML and CSS, we will not be covering these two topics during the course. Students are expected to complete General Assembly's Dash - an online tool that teaches basic HTML, CSS, and JavaScript through projects in the browser.

Additionally, students must complete the following before the start of class:

  1. Complete all of three of the lessons in CodeAcademy's JavaScript Fundamentals track.
  2. Complete CodeSchool's Try Git exercise. This will introduce you to git and GitHub, two technologies we'll be using to store our code and collaborate with others during the course.
  3. A document outlining the pre-course checklist can be found here.

Before the first day of class, please complete the following:

1. Sign up for GitHub 2. Complete The Prework 3. Install Sublime Text 3 4. Install Google Chrome

GRADUATION REQUIREMENTS

In order to successfully complete JSD, students must:

  • Submit 80% of homework assignments
  • Miss no more than three classes
  • Successfully complete and submit a final project

HOMEWORK

Homework will be assigned at the end of class every Wednesday and is due the following Monday. Files and materials for homework will be distributed during class; they are also available through this website.

Students will push (upload) homework to their respective Github repositories for feedback and grading.

NOTE: while late homework will be accepted, it will not receive feedback.
