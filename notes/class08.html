Document Object Model

DOM: Intro

• DOM: Document Object Model

• Connects HTML/CSS to JavaScripts

• "Map" of HTML nodes created by the browser

• Nodes are objects that JS can access/manipulate

DOM: Intro

• Each browser parses HTML differently

• Different browsers mean different DOM's

• Access the DOM using the document object

• To open your console: command + option + j

• Explore nodes using the "Inspector" tool (or, the "Elements" tab)

DOM: Intro

• The DOM provides JS with an API to interact with it

• Since the DOM is an object, we can get/set data!

• Two of the most useful methods relate to querying:

Document.querySelector(String selector)

Document.querySelectorAll(String selector)

• Each returns an element (or group of) from the DOM

Working with the DOM

Working with the DOM

• Open dom_intro in Sublime Text

• There are three ways to inject JavaScript into HTML:

1) Inline Javascript

2) Embeded <script> tags

3) Linked external scripts

Working with the DOM

• Like inline CSS, JavaScript can be executed inline:


  <p onclick="myFunction()">Click Me!</p>

• Please avoid!

• Separate presentation (HTML) from your logic (JS)

• "Separation of concerns"

• Easier to maintain the view and logic components

Working with the DOM

• JS can also run between <script> tags:


  <p>A really exciting paragraph</p>

  <script>
    var num = 5;

    function addTwo (number) {
      return number + 2;
    }

    var newNum = addTwo(num);
  </script>

• Has its uses: plugins and small widgets

• Doesn't allow for reusability across multiple HTML pages

• Cannot cache embeded (or inline) JS

Working with the DOM

• Best practice for injecting JS into HTML is external links

• Use a <script> tag with a src attribute pointing to an external file

• Ensure these tags are placed right before the closing </body>


    // BELOW YOUR HTML!

    <script src="http://jquery.js"></script>
    <script src="react.js"></script>
    <script src="my_app.js"></script>

    // ABOVE THE CLOSING BODY TAG!
  </body>

• Wait until the DOM loads before manipulating it!

• Cannot access nodes the browser hasn't mapped

Work with the DOM

• JavaScript can "listen" for certain events

• A common event: .onclick

• Events capture related data in the first parameter, event

• Assign these events to DOM nodes:


  var btn = document.getElementById('my-btn');

  btn.onclick = function (event) {
    alert('You clicked a button!');
  }

jQuery: What is it?

• jQuery IS JavaScript (source code)

• jQuery is a JavaScript library

• Battle-tested code that increase the speed/stability of development

• Why reinvent the wheel? Use a library instead!

• 1000's of JS libraries, and (for better, or worse) growing daily

jQuery: What is it?

• Released in 2006, currently v3, most widely-used JS library

• Written by John Resig (lives in BK!)

• jQuery's Motto: 'Write Less, Do More'

• Makes working with the DOM + some fn's, easier:

⇒ HTML / CSS manipulation

⇒ Event handling (click events)

⇒ Ajax (...soon!)

• Bonus: Covers JS's browser inconsistencies

jQuery

jQuery

• jQuery in three steps:

1) Use jQuery's CSS selector to query HTML elements

2) Querying returns a jQuery object enclosing the original DOM node

3) Call jQuery methods on the object!

jQuery: CSS SELECTORS

• CSS selectors query HTML for certain elements

• Similar to document.querySelector()

• Use the same selectors you write in CSS!

• The $('x') Function - Get all of the 'X'


  $('div') // select all <div>'s
  $('h2') // select all <h2>'s
  $('p a') // select all <a>'s inside of all <p>'s
  $('p, a') // select all <p>'s and <a>'s

• The $('x') function takes ANY CSS selector

jQuery: CSS Selectors

Query ID's using the # (hashtag):


  $('#someId')
  $('#someId a')

Query Classes using the . (period):


  $('.someClass')
  $('.someClass a')

• Querying classes will return a list, [], of elements

jQuery: Implementing the Library

• To use jQuery (or any library), we must link to it:

⇒ Download and link locally

⇒ Link to a CDN

jQuery: Implementing the Library

Option 1: Download and store jQuery locally

1. Download jQuery from http://jquery.com/download

2. Save the .js file in your project's directory

3. Create a <script> tag ABOVE any of your own JS files

3. Point the src attribute to the the file


  // libs
  <script src="jquery.js"></script>

  // your files
  <script type="text/javascript" src="my_app.js"></script>
jQuery: Implementing the Library

Option 2: Link to an external jQuery source (CDN)

1. Locate an externally hosted version of jQuery:

⇒ https://code.jquery.com/

2. Create a <script> tag ABOVE any of your own JS files

3. Point the src attribute to the jQuery CDN


  // libs
  <script src="https://code.jquery.com/jquery-1.11.3.js"></script>

  // your files
  <script src="my_app.js"></script>
jQuery: Common Functions

• Maniuplate HTML:


  $('div').html('<p>paragraph!</p>');

• Manipulate CSS:


  $('div').css('color', red);

• Get/Set <input> values:


  // Get
  $('input').val()

  // Set
  $('input').val('hello, world!')
jQuery: Common Functions

• Add Event Listeners:


  $('button').click(function (evt) {
    // code here
  })

• Append HTML


  var myValue = 'hello, world!';

  $('div').append('<div>' + myValue + '</div>')
