Functions

Functions

• Chunks of code, grouped together, that perform a certain task

• Reusable statements that can be called anywhere in the program

• Often take an input, perform logic on that input, and return ouput

• Allow the developer to segment large applications into smaller pieces

• What is the name of this function? What does this function do?


  function hello () {
    var greeting = 'hello!';
    greeting = 'hello, world!';
    alert(greeting);
  }
Functions

•A function has the following syntax:

⇒ function keyword

⇒ Name: used to invoke or call the function

⇒ Parameters: options to change the function's input

⇒ Body: code that runs when the function is called


  function speak (words) {
    console.log(words);
  }
Functions

• Remember: you must call functions to use them!

• Calling a function will execute the function's body

• To call a function, follow the function's name with ()


  // function declaration
  function helloWorld () {
    console.log('hello, world!');
  }

  // will not call the function; returns the definition
  helloWorld

  // use () to call a function
  helloWorld()

Function Arguments

• If a function performed the same operation every time, it wouldn't make a dynamic codebase

• We would have to write a new function for small variations

• If we wanted functions that alerted the cast of Point Break (1991):


  // Bad idea...
  function alertBodhi () {
    alert('Patrick Swayze');
  }

  function alertJohnny () {
    alert('Keanu Reeves');
  }
Function Arguments

  function alertBodhi () {
    alert('Patrick Swayze');
  }

  function alertJohnny () {
    alert('Keanu Reeves');
  }

  function alertPappas () {
    alert('Gary Busey');
  }

  function alertTyler () {
    alert('Lori Petty');
  }

• Only 37 more functions to write...

• Does this approach scale?

Function Arguments

• Function parameters remedy this problem by allowing us to call the same function with different values:


  function alertName(name) {
    console.log(name);
  }

  alertName('Keanu');
  alertName('Gary');

  // 'Keanu'
  // 'Gary'
Function Arguments

Arguments: inputs that are "passed" into a function

• Define a function that will accept an argument:


  function hello (name) {
    alert("Hello, it's " + name);
  }

• Call a function that accepts an argument:


  hello('me');
  // "Hello, it's me"

  hello('Chandler');
  // "Hello, it' Chandler"
Function Arguments

• Arguments are accessible in the body using their respective names:


  function hello (name) {
    alert("Hello, it's " + name);
  }

• Whatever is "passed" as name replaces instances of name in the body:


  hello('Chandler');

  // translates to:

  function hello ('Chandler') {
    alert("Hello, it's " + 'Chandler');
  }
Function Arguments: Multiple Arguments

• Functions can accept multiple arguments

• Use a comma-separated list to define:


  function add (x, y) {
    return x + y;
  }

• Call a function with multiple arguments:


  add(2, 2);
  // returns 4

  add(2, 5);
  // returns 7

The return Statment

The return statement

• Determines what value a function returns

• Can treat functions as values and assign to variables:


  function sum(x, y) {
    return x + y;
  }

  var z = sum(3, 4);
  // 7

• Note: if absent the function returns undefined

The return statement

• STOPS function execution:


  function speak (words) {
    return words;

    // The following statements will not run:
    var x = 1;
    var y = 2;
    console.log(x + y)
  }
The return statement

• Useful when applied to conditionals:


  function authenticate (user) {
    if (user.authenticated === false) {
      return authenticate(user);
    }

    return allowEntry(user);
  }

Scope

Scope

• Scope is the set of variables and functions your code has access to

• Two levels of scope:

⇒ Global

⇒ Local

• What will the following code log:


  var num = 10;

  function speak () {
    var num = 100;
    console.log(100)
  }

  speak()
Scope: Global Scope

• Variables and functions defined OUTSIDE a function or block

• Public - accessible from all other scopes:


  // global scope
  var myFavActor = 'Keanu Reeves';

  function logFavActor() {
    // local scope, but has access to global scope
    console.log(myFavActor);

    // automatically global
    favFilm = 'Point Break';
  }

• Note: If you omit var, the variable becomes global

Scope: Local Scope

• Variables and functions defined INSIDE a function or block

• Private - cannot be accessed outside the function


  var a = "this is the global scope";

  function myFunction() {
      var b = "this variable is defined in the local scope";
  }

  myFunction();

  console.log(b);
  // ReferenceError: b is not defined
Scope: Creation

• JavaScript has function scope

• Variables defined within a function are local to that function


  function speak(words) {
    // scope created by the speak function
    var message = words + "!";
  }
Scope: Creation

• A new(er) version of JS introduced block scoping

• Scope created by loops and conditionals

• Must use the let keyword


  if (true) {
    let x = 2;
  }

  console.log(x)
  // ReferenceError: x is not defined
